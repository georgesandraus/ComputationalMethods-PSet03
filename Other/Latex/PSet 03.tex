\section*{Question 1}
\begin{enumerate}
    \item[a-b)] I optimized the function $f(x) = x \sin(5x)$ over the interval $[0,10]$ using Brent, Nelder-Mead, and L-BFGS-B algorithms, testing various starting points ($x_0 \in \{0, 2, 5, 8\}$). Because the function oscillates many times in this interval, it contains several local minima, and the true global minimum is located near the upper bound ($x^* \approx 9.743$, $f(x^*) \approx -9.7409$). 
    
    The results show how extremely sensitive local optimizers are to their initial guesses. For instance, starting Nelder-Mead at $x_0=2$ and $x_0=5$ traps the algorithm in the local minima ($x^* \approx 2.21$ and $x^* \approx 4.72$, respectively). When we run the code, R gives a warning that one-dimensional optimization by this method is unreliable, since the "amoeba approach" makes more sense in a multidimensional setting. Interestingly, L-BFGS-B starting at $x_0=2$ takes large steps, jumps across several valleys, and finds the true global minimum, whereas starting it much closer at $x_0=8$ traps it in a local minimum ($x^* \approx 8.48$). 
    
    The most critical finding, however, is that Brent's method (that uses the default R \texttt{optimize} function) fails to find the global minimum, getting stuck at $x^* \approx 5.97$. Even though it evaluates the entire specified bracket $[0,10]$, it assumes the function is unimodal to guarantee a global optimum. Since $x \sin(5x)$ is multimodal, Brent  gets trapped in a local minimum. This illustrates the theory: for highly non-convex functions, no single algorithm is foolproof, making it strictly necessary to run multiple algorithms from various starting points (multi-start approach) to avoid reporting local minima as global solutions. 
\end{enumerate}

\section*{Question 3}
The analytical true value of $E[u]$ is $1/\sqrt{\pi} \approx 0.5641896$.
\begin{enumerate}
    \item[1)] Using Gauss-Hermite quadrature with 15 nodes on each dimension, I obtained $E[u] \approx 0.5488$.
    \item[2)] Using Monte Carlo integration with 1,000,000 simulations, I obtained $E[u] \approx 0.5643$.
\end{enumerate}
\textbf{Conclusion:} The Monte Carlo result is highly accurate, while Gauss-Hermite performs worse in this scenario. Gaussian Quadrature relies on polynomials approximating the function; however, $\max\{X,Y\}$ introduces a point where the function is non-differentiable. This makes polynomials a bad fit, increasing the required nodes, where the random sampling in Monte Carlo overcomes this problem.
